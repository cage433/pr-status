Report Columns:

  If no columns are specified the default report uses: pr, title, author

  Name                  Alias   Notes
  ------                -----   -----
  pull-request          PR      PR number
  title                 T
  author                A
  loc                   LOC     Scala lines added/removed
  num-comments          NC      Comments since mark (or all if no mark)
  creation-date         CD      Date PR was opened
  last-comment-time     LCT     Time of most recent comment
  my-last-comment-time  MCT     Time of your most recent comment
  mark                  MK      Your mark timestamp
  comment               C       One row per comment; shows comment body
  comment-time          CT      Timestamp of comment (use with C or CA)
  comment-author        CA      Author of comment (use with C or CT)
  reviewers             R       Reviewer names; R=none matches PRs with no reviewers
  unresolved (all)      UC      Unresolved review threads (all authors)
  unresolved (human)    UH      Unresolved review threads (human authors only)
  unresolved (ai)       UA      Unresolved review threads (AI authors only)
  last-activity         LA      Days since most recent comment or review activity
  age                   AG      Days since the PR was opened
  draft                 D       Whether the PR is a draft (true/false)
  youtrack-ticket       YT      YouTrack ticket ID (e.g. PROJ-123); MISSING if absent
  youtrack-project      YP      YouTrack project name (e.g. PROJ); MISSING if absent
  youtrack-id           YI      YouTrack numeric ID (e.g. 123); MISSING if absent
  youtrack-state        YS      YouTrack ticket state (requires youtrack-url/token in config)
  workdays              WD      Total workdays logged against the YT ticket in Timely (hours/8); blank if no YT ticket

  Names and aliases are case-insensitive. Unambiguous prefixes also work
  (e.g. 'cre' for 'creation-date').

Long names:

  Append _ to a column name to display its full name as the header instead
  of the short abbreviation, e.g. 'nc_' shows 'NUM-COMMENTS'.

Timely Report Columns (t / timely command):

  Default columns: developer, project, title, hours, month

  Name              Alias   Notes
  ------            -----   -----
  developer         D, DEV  Person who logged the time
  project           P       Timely project name
  title             T       Log entry description (truncated to 50 chars)
  hours             H       Hours logged (summed across matching entries)
  workdays          W, WD   Hours divided by 8
  month             M       Month of the entry, e.g. Mar-26
  day               Y       Date of the entry, e.g. 2026-03-15
  youtrack-ticket   YT      YouTrack ticket ID extracted from title (e.g. PROJ-123); MISSING if absent
  youtrack-project  YP      YouTrack project part (e.g. PROJ); MISSING if absent
  youtrack-id       YI      YouTrack numeric part (e.g. 123); MISSING if absent

  Names and aliases are case-insensitive. Unambiguous prefixes also work
  (e.g. 'dev' for 'developer', 'pro' for 'project').

  When columns are omitted, rows with identical values across all displayed
  columns are merged and their hours summed. For example:
    t developer,project,hours   shows one row per developer+project

Month filter:

  --filter month=mar-26         Only March 2026
  --filter month=mar            Most recent March
  --filter month=1              Current month only
  --filter month=3              Current month plus 2 previous
  (no month filter)             All cached data from 2025-01-01 onwards

  Projects listed in timely-ignore-project: in the config are always excluded.
